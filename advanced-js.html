1. Immutability – Arrays & Objects

Mutating data is the #1 source of hard-to-find bugs in React & large apps.

Golden rule: Never change (mutate) data directly.
Always create a new copy → predictable state, easier debugging, React can optimize renders. 
// AVOID THIS (mutation)
const numbers = [10, 20, 30];
numbers.push(40);           // original array changed
numbers[1] = 99;

// DO THIS (immutable update)
const newNumbers = [...numbers, 40];           // new array
const updated = numbers.map(n => n === 20 ? 99 : n);
// Object – immutable update (most common in React)
const user = { id: 1, name: "Mia", active: false };

const updatedUser = {
  ...user,
  active: true,
  lastLogin: new Date().toISOString()
};

2. ES6+ Features You Really Need

A. let / const – Block Scope

Use const by default. Only use let when you must reassign.

B. Destructuring + Rest / Spread (very frequent in React)


const user = {
  name: "Rim",
  age: 20,
  email: "rim@example.com",
  role: "admin"
};

const { name, age, ...rest } = user;

console.log(name); // "Rim"
console.log(age);  // 20
console.log(rest); // { email: "rim@example.com", role: "admin" }

// React props example
function Button({ children, variant = "primary", ...props }) {
  return <button className={`btn-${variant}`} {...props}>{children}</button>;
}

C. Optional Chaining & Nullish Coalescing (2020+)

const city = user?.address?.city ?? "Unknown";

3. TypeScript – From Chaos to Confidence

TypeScript doesn't make code slower — it prevents hundreds of runtime bugs.
Most serious frontend jobs in 2025–2026 require it. 
type UserRole = "admin" | "editor" | "viewer";

interface User {
  readonly id: string | number;     // cannot reassign
  email: string;
  role: UserRole;
  preferences?: { theme: "dark" | "light" };   // optional
}

function greetUser(user: User): string {
  return `Hello ${user.email} (${user.role})`;
}
Beginner mistakes to avoid:
  • Using any everywhere → defeats the purpose
  • Writing : any just to silence errors
  • Mutating readonly properties
Best practice 2025: Enable strict: true in tsconfig.json

4. Modern Async / Await + Error Handling

async function loadProfile(id: string) {
  try {
    const res = await fetch(`/api/users/${id}`);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const user = await res.json() as User;
    return user;
  } catch (err) {
    console.error("Profile load failed:", err);
    throw err;               // let React Error Boundary catch it
  }
}
Never forget: Always handle errors in async functions.
Unhandled promise rejections → very bad user experience + hard to debug.

5. JavaScript Patterns Every React Developer Needs

React 18/19 Ready
  • Conditional rendering with && and ternaries
  • ?? nullish coalescing for default values
  • Early returns + guard clauses
  • Key prop in lists (string or number – never index alone)
  • Spreading props {...rest}
  • useState + functional updates when depending on prev state

6. Clean Code & Defensive Programming

Good names > clever code.
Small functions (ideally < 20 lines) do one thing well.
  • isAuthenticated > auth
  • fetchUserProfile > getData
  • Extract complex logic into named functions
  • Fail fast – validate early (guard clauses)
  • Add JSDoc / TS comments for public APIs

/**
 * Greets a user by name and role.
 * @param {Object} user - The user object.
 * @param {string} user.name - The user's name.
 * @param {string} user.role - The user's role (admin, editor, viewer).
 * @returns {string} A greeting message.
 */
function greetUser(user) {
  return `Hello ${user.name} (${user.role})`;
}
