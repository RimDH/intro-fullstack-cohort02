1. React Ecosystem Overview (2026 Perspective)

React is still the dominant frontend library — but the ecosystem has matured.

  • Language: TypeScript is now expected in 85%+ of job postings
  • State Management: Zustand / Jotai / Redux Toolkit → most teams avoid full Redux
  • Routing: React Router v6 / App Router (Next.js)
  • Styling: Tailwind CSS + custom styles
  • Data Fetching: TanStack Query (React Query), SWR, Next.js fetch + caching
  • Meta-Frameworks: Next.js 15 (App Router + Server Components)
For learning fundamentals → pure React + TypeScript + Vite is still the best starting point.
Then move to Next.js once comfortable.

3. Props & State (TypeScript Edition)

Props – Data flowing down

interface ButtonProps {
  label: string;
  onClick: () => void;
  disabled?: boolean;
  variant?: "primary" | "secondary";
}

function Button({ label, onClick, disabled = false, variant = "primary" }: ButtonProps) {
  return (
    <button
      className={`btn btn-${variant}`}{...{ disabled }}
      onClick={onClick}
    >
      {label}
    </button>
  );
}

State – Data living inside component

import { useState } from 'react';

function Toggle() {
  const [isOn, setIsOn] = useState<boolean>(false);

  return (
    <button onClick={() => setIsOn(prev => !prev)}>
      {isOn ? 'ON' : 'OFF'}
    </button>
  );
}

4. Essential React Hooks (The Ones You Use Daily)

useState – Local mutable state

const [name, setName] = useState<string>("");

useEffect – Side effects (fetch, subscriptions, cleanup)

useEffect(() => {
  const timer = setInterval(() => console.log("Tick"), 1000);
  return () => clearInterval(timer); // cleanup!
}, []); // empty = run once on mount

useRef – Persist values across renders + DOM refs

const inputRef = useRef<HTMLInputElement>(null);
useEffect(() => { inputRef.current?.focus(); }, []);

useContext – Access context without prop drilling

const theme = useContext(ThemeContext);

useReducer – Complex state logic

const [state, dispatch] = useReducer(reducer, initialState);

useMemo / useCallback – Performance optimization

const expensiveValue = useMemo(() => computeExpensive(items), [items]);
const stableHandler = useCallback(() => doSomething(id), [id]);
Rule of thumb: Start with useState + useEffect.
Add others only when you hit real problems (re-renders, prop drilling, complex logic).

5. Conditional Rendering & Lists

Conditional Rendering Patterns

{isLoading && <Spinner />}
{error && <ErrorMessage message={error} />}
{!isLoading && !error && <Content />}

// Ternary (good for small conditions)
{isLoggedIn ? <Dashboard /> : <Login />}

// Early return (cleanest for complex logic)
if (isLoading) return <Spinner />;
if (error) return <Error />;

Rendering Lists (always use key!)

<ul>
  {users.map(user => (
    <li key={user.id}>{user.name} – {user.email}</li>
  ))}
</ul>
Never use array index as key unless list is static & never reorders.
Bad: key={index} → bugs on sort/filter/insert

6. Best Practices & Common Beginner Mistakes (2026)

  • Use TypeScript interfaces/props for every component
  • Prefer fragments <></> over unnecessary <div>
  • Always handle loading + error states
  • Put key on the outermost element in .map()
  • Use functional updates in setState when depending on prev
  • Clean up effects (return function from useEffect)
