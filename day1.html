Evolution from Static HTML to Modular SCSS Systems

01. HTML5: The Living Standard

HTML is the Structure. It tells the browser what is a heading, a paragraph, or a button.

Key Concepts

  • Elements: Tags like <div> and <p>.
  • Attributes: src, href, alt, id, class.
  • Nesting: Parent and child relationships.
<body>
  <h1>Hello World</h1>
  <p>This is structure.</p>
</body>

HTML5: Semantic SEO

Experienced devs don't use <div> for everything. They use Semantic Tags.

Pro Tip: Using <main>, <section>, and <article> improves SEO and helps screen readers (Accessibility) navigate your site. 
<!-- SEO Friendly Structure -->
<header>...</header>
<main role="main">
  <section id="hero">...</section>
  <aside>Sidebar Info</aside>
</main>
<footer>...</footer>

02. CSS3: The Art of Styling

CSS turns raw HTML into a visual experience. It controls color, spacing, and typography.

The Box Model

Every element is a box. Understanding Margin, Border, Padding, and Content is vital for any developer.

.box {
  width: 300px;
  padding: 20px; /* Inside space */
  border: 1px solid cyan;
  margin: 10px;  /* Outside space */
}

Mastering Flexbox

Flexbox is a 1D layout tool (Rows OR Columns). It solves the "centering" problem.

.parent {
  display: flex;
  justify-content: space-between; /* Horizontal */
  align-items: center; /* Vertical */
  flex-wrap: wrap; /* Multi-line support */
}
.child { flex: 1; /* Grow to fill space */ }
Exp. Note: Use gap: 20px; on the parent instead of margin on children for cleaner spacing.

CSS Variables & Maintainability

For large projects, hardcoding colors is a disaster. Use Custom Properties.

:root {
  --primary-color: #64ffda;
}

.button {
  background-color: var(--primary-color);
  transition: filter 0.3s ease;
}
.button:hover { filter: brightness(1.2); }

EM vs. REM: Relative Typography

Moving away from fixed px allows your layout to be accessible and scale based on user settings.

1. REM (Root EM)

Relative to the Root (<html>) element. Usually 1rem = 16px. 

html { font-size: 16px; }

.box {
  padding: 2rem; /* Always 32px */
  font-size: 1.5rem; /* Always 24px */
}
Best for: Global spacing, margins, and consistent font sizes.

2. EM

Relative to the Parent's font size. It compounds if nested.

.parent { font-size: 20px; }

.child {
  font-size: 2em; /* 40px */
  padding: 1em;   /* 40px (relative to 40px) */
}
Best for: Component-specific scaling (e.g., icons inside buttons).

Responsive Design: Media Queries

Design for Mobile First. Use Media Queries to "interrupt" the layout as the screen grows.

The Mobile-First Approach

Write your base CSS for small screens, then add complexity for Desktop using min-width. 

/* Base: Mobile (1 Column) */
.container { display: block; }

/* Desktop: 768px and up */
@media (min-width: 768px) {
  .container { 
    display: flex; 
    gap: 20px;
  }
}

SCSS: Nesting & Variables

SCSS makes your stylesheets readable by mirroring your HTML structure and centralizing values.

Nesting & Parent Selectors

Use & to reference the parent element for pseudo-classes like hover.

$primary: #64ffda;
$radius: 12px;

.nav-card {
  background: #112240;
  border-radius: $radius;
  
  h2 { font-size: 1.5rem; } // Nested child
  
  &:hover { 
    border-color: $primary; 
  }
}

SCSS: Functions & Logic

Functions allow you to perform calculations and return values, keeping your CSS dynamic.

// Define a function for PX to REM
@function toRem($px) {
  @return ($px / 16) * 1rem;
}

.hero-title {
  font-size: toRem(64); // Becomes 4rem
  width: 100% / 3;      // Math operations
}

.card {
  // Use built-in color functions
  color: lighten($primary, 20%);
}

Why use Functions?

  • Consistency: Convert units automatically.
  • Math: Handle grid widths without a calculator.
  • Color Manipulation: Create hover states automatically with darken() or opacify().

Professional SCSS Architecture

The 7-1 Pattern: 7 folders, 1 main file. It ensures your project stays organized as it grows.

  • abstracts/: Variables, mixins, and functions.
  • base/: typography, and resets.
  • components/: One file per UI piece (button.scss, card.scss).
  • layout/: Global structural pieces (header, footer, grid).
// main.scss - The orchestrator
@import 'abstracts/variables';
@import 'abstracts/functions';
@import 'base/reset';
@import 'components/navbar';
@import 'components/footer';

// Browser only sees the compiled main.css!

04. Bootstrap 5: The UI Toolkit

Bootstrap is a framework that provides pre-built components and a responsive grid.

Key Update: Bootstrap 5 no longer requires jQuery! It is much lighter and uses Vanilla JS.

It includes: Modals, Navbars, Tooltips, Buttons, and Carousels.

The 12-Column Grid System

Bootstrap divides the screen into 12 parts. You decide how many parts each element takes at different screen sizes.

<div class="container">
  <div class="row">
    <div class="col-12 col-md-8">Main (Full on mobile, 8/12 on Desktop)</div>
    <div class="col-12 col-md-4">Sidebar (Full on mobile, 4/12 on Desktop)</div>
  </div>
</div>

Bootstrap: Utility-First Thinking

Modern Bootstrap encourages using "Utilities" instead of writing new CSS for small changes.

  • d-flex justify-content-center (Flexbox layout)
  • mt-5 mb-2 (Margin Top/Bottom)
  • text-uppercase fw-bold (Font styling)
  • rounded-pill shadow-lg (Decoration)
